# Chronica Clients

このフォルダには、Chronica MCPサーバーと連携するクライアントアプリケーションを配置します。

## 現在のクライアント

### `client_gemini.py`
- **状態**: アーカイブ（参考用）
- **説明**: Google Gemini APIを使用したクライアント
- **問題**: 無料枠の1日20リクエスト制限により実用的でない
- **用途**: 将来の参考用として保持

## 今後の予定

### `client_ollama.py` ✅ 実装済み
- **説明**: Ollamaを使用したローカルLLMクライアント
- **メリット**: 
  - API制限なし
  - プライバシー保護
  - オフライン利用可能

### 日本語品質向上戦略

Ollamaの日本語品質を向上させるため、以下の戦略を採用：

1. **推奨モデル**
   - **Qwen 3 (8B)** - 最優先推奨 ⭐
     - Qwen 2.5の後継モデルで、性能が大幅に向上
     - 多言語対応の強化（119言語）
     - 長いコンテキストウィンドウ（32K）
     - 思考モード（Thinking mode）サポート
     - JSONの書き間違いが少なく、Tool呼び出しに最適
     - 日本語も非常に上手
   - **Qwen 3 (4B)** - リソースが限られている場合
     - 軽量で高速
     - 8Bより性能は劣るが、十分に実用的
   - **Qwen 3 (30B)** - 最高性能が必要な場合
     - 最大の性能を発揮
     - 高いVRAMが必要
   - **Gemma 3 (4B)** - 代替選択肢
     - Google純正、文脈を読む力が強い
     - ChronicaのようなパートナーAIに最適の情緒
   - **ELYZA Llama-3** - 日本語会話重視の場合
     - 日本語特化の最高峰、会話は最強
     - Tool指示ではQwen 3に一歩譲る

2. **Chronica構造化情報の活用**
   - 時間、記憶、コンテキストを自然な日本語に変換
   - テンプレートエンジンとの組み合わせ

3. **プロンプトエンジニアリング**
   - Tool呼び出しの正確性を重視したシステムプロンプト
   - Few-shot Learningの活用

詳細は `docs/japanese_quality_strategy.md` を参照してください。

## 使用方法

各クライアントは独立して動作します。MCPサーバー（`run_server.py`）が起動している必要があります。

```powershell
# Ollama版
# 事前にモデルをダウンロード（推奨: qwen3:8b）
ollama pull qwen3:8b

# クライアントを起動
python clients/client_ollama.py
```
