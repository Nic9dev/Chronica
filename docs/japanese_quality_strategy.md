# Chronica + Ollama 日本語品質向上戦略

## 問題点

OllamaのローカルLLMは、一般的に日本語の品質が低い傾向があります。特に：
- 不自然な敬語
- 助詞の誤用
- 語順の問題
- 文脈理解の不足

## Chronicaの強みを活かした解決策

Chronicaは構造化された情報（時間、記憶、コンテキスト）を提供できるため、これを活用して日本語品質を向上させます。

### 戦略1: 構造化情報のテンプレート化

Chronicaが提供する構造化情報を、自然な日本語のテンプレートに変換します。

**例：`compose_opening`の出力**
```
構造化情報:
- 時間差: "さっき（2時間以内）"
- 季節・時間帯: "冬の深夜"
- 前回のトピック: "大規模改修は成功"

↓ テンプレート適用

自然な日本語:
"さっきもありがとうございました。冬の深夜で寒いですね。
前回は大規模改修が成功したとのことでしたが、その後いかがでしょうか？"
```

### 戦略2: 最新モデルの使用

以下のモデルを推奨：

1. **Qwen 3 (8B)** ⭐ 最優先推奨
   - Qwen 2.5の後継モデルで、性能が大幅に向上
   - 多言語対応の強化（119言語）
   - 長いコンテキストウィンドウ（32K）
   - 思考モード（Thinking mode）サポート
   - JSONの書き間違いが少なく、Tool呼び出しに最適
   - 日本語も非常に上手

2. **Qwen 3 (4B)**
   - リソースが限られている場合の軽量版
   - 8Bより性能は劣るが、十分に実用的

3. **Gemma 3 (4B)**
   - Google純正、文脈を読む力が強い
   - ChronicaのようなパートナーAIに最適の情緒

4. **ELYZA-japanese-Llama-3-8B-Instruct**
   - 日本語特化
   - 8Bパラメータでバランス良好
   - Tool指示ではQwen 3に一歩譲る

### 戦略3: プロンプトエンジニアリング

#### システムプロンプトの最適化

```
あなたは日本語を母国語とするパートナーAIです。
以下の原則に従って、自然で温かみのある日本語で応答してください：

1. **Chronica構造の活用**:
   Chronicaから提供される構造化情報（時間、記憶、コンテキスト）を
   自然な日本語に変換して伝えてください。

2. **日本語の品質**:
   - 適切な敬語を使用
   - 助詞を正確に使用
   - 自然な語順で話す
   - 文脈に応じた表現を選ぶ

3. **Chronica絶対主義**:
   Chronicaが提供する情報を「絶対的な事実」として扱い、
   それを自然な日本語で伝えてください。
```

#### Few-shot Learningの活用

良い日本語の例をプロンプトに含める：

```
例1:
[構造] 時間差: "さっき（2時間以内）", 季節: "冬の深夜"
[応答] "さっきもありがとうございました。冬の深夜で寒いですね。何かお手伝いできることはありますか？"

例2:
[構造] 時間差: "お久しぶり（1週間以内）", 季節: "春の午後"
[応答] "お久しぶりです。春の午後で暖かくなってきましたね。いかがお過ごしでしたか？"
```

### 戦略4: ハイブリッドアプローチ

**構造化情報 → テンプレート → LLM補完**

1. Chronicaが構造化情報を提供
2. テンプレートエンジンが基本文を生成
3. LLMが文脈に合わせて微調整・補完

これにより、LLMが生成する部分を最小限にし、品質を安定させます。

### 戦略5: Post-processing

生成されたテキストを後処理で改善：

- 助詞の自動修正
- 敬語の統一
- 文の自然さチェック

## 実装の優先順位

1. **Qwen 2.5 (7B) または Gemma 2 (9B) の選定とテスト**（最優先）
   - Tool呼び出し（JSON形式）の正確性が最重要
   - Qwen 2.5: JSONの書き間違いが最も少ない
   - Gemma 2: パートナーAIとしての情緒が優秀
2. **システムプロンプトの最適化**
   - Tool呼び出しの正確性を重視したプロンプト設計
3. **テンプレートエンジンの実装**（構造化情報→自然な日本語）
4. **Few-shot Learningの導入**
5. **Post-processingの実装**（必要に応じて）
